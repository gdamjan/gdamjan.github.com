3 ways to IRC

2020-10-10

I've been using IRC for quite some time now, mainly to keep in touch with the global and local free-software communities and the people around the Skopje Hacklab KIKA (#lugola on freenode). Even at this age IRC is still used by many projects as one of the main communication platforms.

IRC is an old protocol, started its life in 1988, so using it in this age has its challenges. First of all, irc is a simple plain text protocol that assumes an always-on persistent TCP connection. If the connection is broken, you vanish from the network, and when you reconnect you've lost the discussion in the meantime. This is especially problematic today, when we expect to roam between wifi, mobile and wired networks, or put our laptops and mobile phones to sleep.

There are some efforts in the IRCv3 community to extend and improve the protocol, by allowing for capability negotiation, message ids, replay of missed messages since the last id seen, and keeping a buffer for each user. But major irc networks like freenode haven't implemented those extensions yet, and considering the added cost per user, they probably never will.

So these are the 3 ways that I use to connect to IRC from my laptop, phone and tablet, without having to compromise and miss "important" chat messages:

IRC on mobile over matrix

Matrix is a secure and advanced chat protocol designed for modern times. It uses http as transport but has strong support for reconciling the client and server state. In simple words, you don't miss chats when roaming or sleeping. Matrix has support for gateways to other chat networks, and one of those is the irc gateway. We (hacklab kika) host our own matrix server and irc gateway at spodeli.org, so I use that (find me at @gdamjan:spodeli.org). I use the Android app, also available on F-Droid, called Element that connects to the matrix server, the matrix server connects to the irc gateway (they are on the same server), and then the irc gateway connects to freenode. Since the server can easily maintain a persistent TCP connection to freenode, there's no loss of data or chat history.

Additionally Matrix also has a couple of desktop clients, and a web app interface, so I can use that in parallel in any modern browser. The matrix protocol, and its federated network is becoming very popular in the last years, replacing irc for organizations like Mozilla, and being implemented in governments throughout the world.

IRC on the desktop over znc

On my desktop/laptop I use the communi built with Qt/QML. I like the minimalist look of communi and the quick navigation in the app.

While communi does support IRCv3 extensions, freenode doesn't, so that doesn't fix the issues. The solution: I also run a znc bouncer on a VPS, where it can maintain the persistent TCP connection to freenode. Znc and the playback plugin, support enough of IRCv3 to replay any missed messages when my laptop goes from one wifi network to another (or to a hotspot), or if my ADSL service reconnects, or my laptop is put to sleep, etc. As an added benefit, znc keeps my freenode registered nickname online at all times.

IRC in a web app

Sometimes I need to connect to irc from ad-hock computers. For example my experimental laptop with (currently) Fedora Silverblue which I haven't bothered to install communi on. Or my ChromeOS tablet which might not even support applications as such (ok - in truth it does support Crostini, a VM with containers on top of ChromeOS, it's a bit too much for an irc client). Or my parents computer where I only have a browser.

The same vps that runs my znc also runs the lounge, a nodejs web app. Again, it can maintain the persistent TCP connection to freenode, and it keeps the buffer of unread messages for me when I'm offline. Open the web app from anywhere you get your chats, history, etc. Thelounge is better looking than the matrix web app Element, for now.

My lounge account is configured to connect to my znc, so no matter if I'm on my laptops communi, or in the ChromeOS browser I always read and write from the same user. But that's fully optional.

End note

Hosting your own matrix, irc gateway, znc or thelounge servers might be complicated and time consuming, and even expensive just to get reliably on irc. If you ask nicely on #lugola on freenode, you can get an account on any of these services we host :)
